# Agent Notes

- Prefer the skills in `.agents/skills/` before inventing a route-tool workflow.
- Use mock evals before real provider calls.
- Do not add secrets, API keys, route-session private data, or customer route exports.
- Keep `SKILL.md` files concise; put detailed examples in root `references/`.
- When adding tools or changing tool semantics, update `references/tool-contracts/route-tools.md` and at least one eval case.
